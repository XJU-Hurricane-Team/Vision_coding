# 网页本地版

### 运行程序

- project_forlder

  - python app.py

- 点击显示在终端的对应电脑端网址进入网页端界面

- 在手机上输入对应手机端网址进入手机GPS经纬度传输界面

  - 二者需要使用共同的wifi（共用同一个内网）

- **识别情况简介**

  - 种类

  - ```
    Crack:          裂纹
    Manhole:        井
    Net:            网
    Patch-Crack:    裂纹块
    Patch-Pothole:  坑洼块
    Pothole:        坑洼
    
    names:
      0: Crack
      1: Manhole
      2: Net
      3: Pothole
      4: Patch-Crack
      5: Patch-Net
      6: Patch-Pothole
      7: other
      8: Other
    ```
    
  - 严重程度
  
  - ```
    轻度损伤 (1-2条)  --  黄色
    中度损伤 (3-5条)  --  橙色
    严重损伤 (≥6条)   --  红色
    ```

### 说明

- **一.能实现的程度**
  - 1.根据定位获取当前位置的静态地图
  - 2.能够根据道路损伤识别情况，在静态地图上做对应标识
  - 3.能够记录下损伤严重的路面情况并能够通过查询该路面的经纬度得到该位置的静态地图



- **二.已经解决的问题**

  - 1.定位问题
  - [ ] 使用电脑网页端口的ip
  - [x] 使用同一个内网的手机GPS定位
  - [ ] 使用硬件设备（需要用这个）

  - 2.移动问题
    - 使用每秒钟获取定位信息（经纬度）后通过Web墨卡托投影方法换到静态地图上对应的位置
  - 3.裂缝识别重复问题

  - [ ] 使用yolo识别的ID命名区分
  - [x] 在静态地图上使用范围划分



- **三.需要优化的问题**
  - 1.需要更新成真的网页端
  - 2.需要更新成硬件设备定位
  - 3.需要测试定位移动同步
  - 4.需要扩展到硬件设备上使用
    - 可以考虑划分功能
      - 硬件端负责解决现场识别，上传识别结果
